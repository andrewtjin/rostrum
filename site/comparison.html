How Rostrum compares

Verbatim and most in-round tools are VBA .dotm macros. Rostrum is a modern Word add-in. That difference is not cosmetic — here is what it buys you, claim by claim.

At a glance

  Rostrum (Office.js add-in) Verbatim & VBA macros
Time to hide a real tournament case file ~4–5 seconds — measured on real 35- and 49-card case files ~5–10 minutes on the same files
While it works through a long document Word stays responsive — switch to your other open documents and keep working Runs on Word's UI thread, so the whole app is frozen until it finishes
Recovering hidden text without the tool Native — Word's own Font dialog (Ctrl+D); no add-in required Macro-applied effects generally need the macro (and its template) to reverse
Reversing shrink / condense Lossless — rebuilt from markers inside the document itself Not guaranteed to reconstruct from the document alone
Opening a processed file on another machine The same view re-derives from data inside the file Depends on the recipient having the template installed

Keep working during invis mode

A VBA macro runs synchronously on Word's own UI thread: while it churns through a long file, the entire application is frozen — you can't scroll, type, or switch documents until it finishes. Rostrum's Invisibility Mode is an add-in running in a separate process, so a hide never locks the host.

Kick off a hide on a huge case file and you can immediately switch to your other open documents and keep reading, editing, and prepping while it runs — you can even start Rostrum on a second document. That is a genuine multi-document, in-round workflow a macro cannot offer. Even the heaviest docs stay non-blocking: a 900k-word file hides in well under a minute, and Word stays responsive the entire time.

Reversible without the add-in

Invisibility Mode hides text with Word's Hidden font attribute — the exact attribute Word's own Font dialog writes. So a recipient who doesn't have Rostrum can still recover everything:

  1. Select the text (Ctrl+A for the whole document).
  2. Home ▸ Font dialog (Ctrl+D) ▸ clear the Hidden checkbox — or toggle Home ▸ ¶ to view hidden text.

The guarantee lives in the file format, not the tool: nothing Rostrum hides is locked behind Rostrum.

Lossless shrink & condense

Rostrum's Condense & Shrink is the lossless answer to the usual shrink/condense pass. Shrink steps the card's non-underlined text down a font size while keeping the underlined cut, highlights, cites, and headings full-size; Condense collapses paragraph spacing.

Both are fully reversible from the document alone. Condense records what it merged in self-describing markers embedded in the file (no sidecar), so Uncondense and Unshrink reconstruct the original — even on a machine that never had the add-in.

The same view on every machine

When you hide a doc, Rostrum's on-state and your keep-colors are written into the file as a single custom XML part. There is no per-span tracking and no bloat, and it survives format round-trips.

The upshot: any machine that opens the file re-derives the same view deterministically. Share a hidden doc and your partner sees exactly what you see — they need nothing installed to read it, and Rostrum to re-hide it the same way.
