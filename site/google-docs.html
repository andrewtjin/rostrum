Rostrum for Google Docs

A modern debate tool suite, built for debaters by debaters.

You paste one script into the document's own Apps Script project, and a Rostrum menu appears after a reload. The whole process takes about two minutes.

Version v__GDOCS_VERSION__. This early release covers Hide, Show All, Apply debate styles, and Mark cite. Full usage guide: gdocs/README.

What "Hide" actually does. Hide reduces every word of body text except headings, cites, and highlighted text to a 1‑point size, so a long file reads like a speech doc and the page count collapses. It does not conceal anything: anyone can restore the text with Select All, then font size 11 (see Reveal everything below). Show All restores every word. This matches how Rostrum behaves in Word.

1 · Get the two files

You paste in two small files: Code.gs (the script) and appsscript.json (the permissions manifest).

Code.gs Download didn't start? Use the direct link.

appsscript.json the permissions manifest (the Docs service plus two scopes).

2 · Paste it in

You install per‑document, into the document's container‑bound script. Nothing is published anywhere; the script lives inside the document.

  1. Open the Google Doc you want Rostrum in.
  2. Extensions → Apps Script. A new editor tab opens with an empty Code.gs.
  3. Select everything in Code.gs and delete it, then paste the entire contents of the downloaded Code.gs in its place.
  4. Tell the project which permissions it needs (a one‑time manifest step): click Project Settings (the gear, left rail) and tick "Show 'appsscript.json' manifest file in editor." Back in the Editor, open appsscript.json and replace its contents with the downloaded one. (Alternatively, add the Docs API advanced service directly: Editor → Services (＋) → Google Docs API → Add.)
  5. Save (Ctrl/⌘+S).
  6. Reload the Google Doc tab. After the reload a Rostrum menu appears in the menu bar, next to Help.

✓ A Rostrum menu now sits in the document's menu bar.

3 · First run: authorize, then run again

The first time you choose a Rostrum action, Google interrupts to ask for permission, so your first click is used by the consent flow rather than the action itself.

  1. Choose Rostrum → Hide (or any action). A Google authorization dialog appears.
  2. Choose your account.
  3. Because this is your own unverified script, Google shows a warning screen. Click Advanced, then "Go to (your doc) (unsafe)." This is expected for a personal script you pasted yourself; you are granting it access to your own document.
  4. Review the permissions and click Allow.
  5. The dialog closes without running the action, which is normal. Choose the same Rostrum action again and it runs.

Rostrum runs as the account you authorized, which is the account signed into this Chrome profile when you click Allow. On a managed or school (Workspace Education) account, administrator policy may block consent entirely; if so, install on a personal Gmail account instead.

Reveal everything without Rostrum

If you (or anyone you send the document to) ever need the text back without Rostrum installed:

Select All (Ctrl/⌘+A), then set the font size to 11. Everything becomes visible again.

Sharing with teammates

A Google Doc's bound script travels with a copy. To give a teammate the styled, Rostrum‑enabled document, use File → Make a copy (or share a copy). The copy carries the bound script, so they get the Rostrum menu after their own first‑run authorization. There is no central install to manage. (Shared copies are not counted as downloads; see Privacy.)

Team norm: don't edit a document while it is hidden. Run Show All first. Moving or copying hidden text can separate it from its saved sizes; Show All still brings it back (with a note), but editing while visible is cleaner.

Limits in this version
  • Single‑tab documents only. A document that uses Google Docs tabs is not supported; Rostrum stops and explains why instead of acting on the wrong content.
  • Suggesting mode: Hide and Apply debate styles do not run while there are unresolved suggestions; Show All always works.
  • Cut and paste of hidden text can separate it from its saved sizes. Show All then normalizes it back to full size and reports how many passages it reset.
  • Condense, Shrink, and Marketplace publication are not part of this version.
No tracking inside Google Docs. The pasted script only ever changes font size and named‑style definitions; it sends nothing back as you use it. There is no document content, no personal data, and no usage reporting. The single anonymous number Rostrum records is the one‑time Code.gs download from this site (and only through the counted link above, not the direct copy). Full disclosure: Privacy.
