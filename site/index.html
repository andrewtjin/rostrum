Rostrum

A Word add-in that hides debate-card body text while keeping headings, cites, analytics, and highlighted runs — and is natively reversible, even without the add-in.

Download manifest.xml

You install the add-in by registering this one file in Word — once. Rostrum runs from this site; you host nothing and run no server. Windows registers it as a trusted add-in catalog (steps below); Mac drops it into Word's wef folder.

Desktop Word only. Rostrum hides text with Word's Hidden font attribute (WordApiDesktop 1.2), which exists on Word for Windows and Mac but not Word for the web. On an unsupported host the add-in explains the manual fallback instead of failing silently.

Install (sideload)

Pick your platform. After the one-time setup, Rostrum appears under Insert ▸ My Add-ins and adds a Rostrum tab to the ribbon.

Windows — register the add-in (one-time)

Rostrum installs as a trusted add-in catalog: a shared folder holding manifest.xml. Once registered, the Rostrum tab appears on the ribbon of every document automatically — new and existing — with no per-document re-adding, and it survives restarting Word.

  1. Save manifest.xml into a folder, e.g. C:\Users\you\Documents\rostrum-addin.
  2. Right-click that folder ▸ Properties ▸ Sharing ▸ Share… and share it with yourself. Copy the network path (it looks like \\YOUR-PC\rostrum-addin). This one-time share is what lets Word load Rostrum on every document.
  3. In Word: File ▸ Options ▸ Trust Center ▸ Trust Center Settings ▸ Trusted Add-in Catalogs.
  4. Paste the share path into Catalog Url, click Add catalog, tick Show in Menu, then OK.
  5. Restart Word.
  6. Insert ▸ My Add-ins ▸ Shared Folder ▸ Rostrum ▸ Add. The Rostrum tab is now on the ribbon of every document.

Updating: when a release changes the ribbon, replace the manifest.xml in your shared folder with the latest and restart Word. Behavior-only fixes are served live from this site — just restart Word.

Mac

  1. Copy manifest.xml into this folder (create the wef folder if it doesn't exist):
    ~/Library/Containers/com.microsoft.Word/Data/Documents/wef/
  2. Restart Word.
  3. Insert ▸ My Add-ins ▸ Rostrum. (If you don't see it, also check the Developer Add-ins group of that menu.)

Updates

Rostrum runs from this site, so you don't reinstall to get fixes. There are two kinds of update, and only one of them needs you to do anything:

  • Fixes & improvements — automatic. Changes to how Rostrum behaves (speed, bug fixes, tweaks to existing buttons) are served live from this site. Just restart Word to pick them up — or, with the Rostrum pane open, click inside it and press Ctrl+F5 to refresh immediately. You re-download nothing.
  • New version — re-download manifest.xml. When a release adds a new ribbon button or tab (or we announce a new version number), the change lives in the manifest file itself — the one file you registered. Your copy is a local snapshot, so grab the latest manifest.xml, replace your saved copy, clear the Office cache (see Uninstall for how), and restart Word.

Version announcements live on the GitHub releases page. Rule of thumb: if a button you expect is missing after an update, re-download the manifest.

Uninstall

Rostrum installs no program — removing it means dropping the manifest registration and clearing Word's add-in cache. Pick your platform.

Windows

  1. Quick remove (current document): Insert ▸ My Add-ins, right-click Rostrum ▸ Remove. To remove it everywhere, continue below.
  2. Close Word.
  3. File ▸ Options ▸ Trust Center ▸ Trust Center Settings ▸ Trusted Add-in Catalogs. Select the Rostrum folder row and untick Show in Menu, then tick Next time Office starts, clear all previously-started web add-ins cache (Word 2108+) ▸ OK.
  4. Restart Word. The Rostrum tab and its My Add-ins entry are gone from every document.

Still lingering, or an older Word without that checkbox? With Word closed, delete everything inside %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\ — note this clears all sideloaded add-ins, not just Rostrum. Ran an older Rostrum installer? Also delete the value ea3fb238-… under HKCU\Software\Microsoft\Office\16.0\WEF\Developer.

Mac

  1. Quit Word.
  2. Delete manifest.xml from the wef folder you installed it into:
    ~/Library/Containers/com.microsoft.Word/Data/Documents/wef/
    (or clear that folder's contents to remove every sideloaded add-in).
  3. Restart Word.

Reading a Rostrum document without the add-in

Hidden text uses Word's standard Hidden font attribute, so anyone can recover it with no add-in installed:

  1. Select the text (Ctrl+A for the whole document).
  2. Home ▸ Font dialog (Ctrl+D) ▸ clear the Hidden checkbox — or toggle Home ▸ ¶ to view hidden text.
